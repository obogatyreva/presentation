What is React?

React is a JavaScript library for building user interfaces.

  • It is used for developing complex and interactive web and mobile UI.
  • It is used as a base in the development of single-page or mobile applications.
  • It composed complex UIs from small and isolated pieces of code called “components”.

History

React was created by Jordan Walke, a software engineer at Facebook. React is influenced by the likes of XHP which is a simple HTML component framework for PHP. It was first deployed on Facebook's newsfeed in 2011 and later on Instagram.com in 2012. It was open-sourced at JSConf US in May 2013.

The advantages of React

Some of the major advantages of React are:

  • SEO-friendly
  • JSX
  • Simplicity
  • UI Test Cases

React has a small API, and you have to understand 4 concepts to get started:

  1. Components
  2. JSX
  3. State
  4. Props

Components

Component is a JavaScript class or function that accepts inputs i.e. properties(props) and returns a React element that describes how a section of the UI (User Interface) should appear.

functional component


							function Welcome(props) {
							  return <h1>Hello, {props.name}</h1>
							}

class component


							class Welcome extends React.Component {
								render () {
							  return <h1>Hello, {this.props.name}</h1>
							}
						}

State is an object.

It is mutable unlike props and creates dynamic and interactive components. it is accessed via this.state(). 


                class Clock extends React.Component {
                  constructor(props) {
                    super(props);
                    this.state = {date: new Date()};
                  render() {
                    return (
                      <div>
                      <h1>Hello, world!</h1>
                      <h2>It is {this.state.date.toLocaleTimeString()}.</h2>
                      </div>
                    );
                  }
                }

Props is shorthand for properties. They are read-only components which must be kept pure i.e immutable. They are data passed down from a parent component to a child component.

In React world, the term "virtual DOM" is usually associated with React elements.

How Virtual DOM helps React?

React maintains two Virtual DOM at each time. One contains the updated Virtual DOM and one which is the pre-update version of this updated Virtual DOM. We can compare the pre-update version with the updated Virtual DOM and find out what has changed in the DOM. This process is known as ‘diffing’.

Difference between Real DOM and Virtual DOM

Real DOM Virtual DOM
It updates slow. It updates faster.
Can directly update HTML. Can't directly update HTML.
Creates a new DOM if element updates. Updates the JSX if element updates.
DOM manipulation is very expensive. DOM manipulation is very easy.
